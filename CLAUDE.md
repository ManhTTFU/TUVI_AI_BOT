# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Tử Vi Đẩu Số (Vietnamese Purple Star Astrology) system: REST API + SSR website. All user-facing copy is Vietnamese. Astrology calculations are delegated to the `iztro` library; AI narration is generated by Deepseek (OpenAI-compatible API) per-section.

## Commands

Package manager is **pnpm** (workspaces). Node ≥ 20.

```bash
pnpm install              # installs deps + auto-runs download-fonts postinstall
pnpm download-fonts       # manually (re)download Noto Sans TTFs into ./fonts
pnpm dev                  # run api + web in parallel
pnpm dev:api              # single app (also: dev:web)
pnpm build                # run `build` in every workspace
pnpm start                # api + web started concurrently via `&`

# Typecheck a single package without emitting:
pnpm --filter @tuvi/ai exec tsc --noEmit
pnpm --filter @tuvi/web exec tsc --noEmit

# Web only
pnpm --filter @tuvi/web build
pnpm --filter @tuvi/web dev   # Next.js on :3100
```

There is no test suite and no linter wired into CI. `next lint` exists in `apps/web` but no ESLint config is committed.

## Env loading (two different mechanisms)

- **API**: `tsx --env-file=../../.env` — loads the **root** `.env` only. Don't add `dotenv` calls; update `.env` at repo root.
- **Web (Next.js)**: Next auto-loads `apps/web/.env.local` from its own cwd; it does **not** read the root `.env`. When adding a new `NEXT_PUBLIC_*` var, update both `.env` (for reference/consistency) and `apps/web/.env.local`.
- Required vars: `DEEPSEEK_API_KEY` (+ optional `DEEPSEEK_BASE_URL`, `DEEPSEEK_MODEL`, `AI_CONCURRENCY`), `API_PORT`, `OUTPUT_DIR`, plus the `NEXT_PUBLIC_*` trio. **Auth/DB block:** `DATABASE_URL` (Postgres / Neon), `AUTH_SECRET`, `NEXTAUTH_URL`, `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET` (or email SMTP block). Casso (optional): `CASSO_ENABLED=false` by default, `CASSO_API_KEY`, `CASSO_WEBHOOK_SECRET` khi bật.

## Auth + Payment architecture

Auth.js v5 với Drizzle adapter sống trong `apps/web/src/auth.ts`. JWT-less session
strategy (`session: { strategy: 'database' }`) — session bảng `sessions`, mỗi request
server đọc thẳng từ DB nên `role` + `balanceVnd` luôn fresh.

Provider:
- Google OAuth (cần `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET`)
- Email magic link qua Nodemailer (cần `EMAIL_SERVER_HOST` + `EMAIL_FROM`)

Tự off provider nếu env thiếu. Promote admin: `pnpm --filter @tuvi/db exec tsx --env-file=../../.env src/promote-admin.ts <email>`.

Payment flow:
1. User vào `/vi-cua-toi` → bấm "Tạo lệnh nạp" → POST `/api/wallet/topup-request` sinh `bankRef` unique → hiển thị QR + thông tin bank + ref code.
2. User chuyển khoản với nội dung CK = bankRef.
3. Admin vào `/admin/transactions` duyệt giao dịch pending → POST `/api/admin/transactions/approve` → atomic update balance + insert log + **publish SSE** event tới user.
4. User trang ví subscribe `/api/wallet/stream` (EventSource) → nhận event `balance` realtime → update UI ngay.
5. Casso webhook `/api/casso/webhook` reconcile tự động khi bật `CASSO_ENABLED=true` (backup path).

Charge flow (lập lá số):
1. User submit form `/xem-tu-vi` → POST `/api/tuvi/submit` → auth check + atomic deduct combo price (default 40k, đọc từ bảng `prices`) + insert tx log + calculate chart (iztro) + save chart vào DB.
2. Trả về `{chartId, slug, chart}` + push SSE balance.
3. FE fire parallel: POST `/api/tuvi/<chartId>/analyze` + POST `/api/tuvi/<chartId>/deep-readings`. Mỗi endpoint check cache theo `birthHash` trước (share giữa user cùng ngày sinh), miss thì gọi AI + save cache.

402 INSUFFICIENT_BALANCE: FE hiển thị banner + link `/vi-cua-toi`.

## Canonical data flow

```
BirthInfo  →  calculateChart (astrology)
           →  summarizeChartForAI (text block fed to LLM)
           →  analyzeChart (ai) — 6 parallel Deepseek calls (Promise.all + p-limit semaphore + retry)
                 sections: overview, career, love, health, decade, advice
                 emits onProgress(step, total, key) for UI feedback (theo thứ tự HOÀN THÀNH, không phải gọi)
           →  buildPdf (pdf)      produces Buffer
           →  saveFullResult + savePdf → output/{charts,pdfs}/<slug>.{json,pdf}
```

`makeChartSlug(name, birthDate)` produces the URL-safe slug used everywhere as primary key. Lookups (`GET /api/tuvi/chart/:slug`, web SSR page) read `output/charts/<slug>.json` from disk — there is no database.

## API surface

### Next.js routes (`apps/web/src/app/api/*` — primary path, frontend uses these)

Auth:
- `GET|POST /api/auth/*` — Auth.js v5 (signin/signout/callback/session)

Tử Vi (paid, auth-gated):
- `POST /api/tuvi/submit` — charge combo 40k, calculate chart, save to DB, return `{chartId, slug, chart}` hoặc 402 `INSUFFICIENT_BALANCE`
- `POST /api/tuvi/<chartId>/analyze` — load chart, check `analyses` cache by birthHash, AI or cached
- `POST /api/tuvi/<chartId>/deep-readings` — same pattern, cache by `(birthHash, year)`

Wallet:
- `POST /api/wallet/topup-request` — sinh bankRef + insert pending tx
- `GET  /api/wallet/balance` — số dư hiện tại
- `GET  /api/wallet/transactions` — lịch sử 50 giao dịch gần nhất
- `GET  /api/wallet/stream` — SSE channel (in-process pub/sub via `lib/sse-bus.ts`)

Admin (role=admin required):
- `POST /api/admin/credit` — cộng/trừ tiền user, publish SSE
- `POST /api/admin/transactions/approve` — duyệt topup, publish SSE
- `POST /api/admin/transactions/reject`
- `POST /api/admin/bank-config` — lưu thông tin bank
- `POST /api/admin/bank-config/upload-qr` — upload QR PNG/JPG/WebP <500KB vào `public/uploads/`

Casso (off by default, `CASSO_ENABLED=true` để bật):
- `POST /api/casso/webhook` — Casso gửi danh sách giao dịch ngân hàng → `reconcileCassoTransactions` match bankRef → auto credit + SSE

### Express endpoints (`apps/api/src/routes/tuvi.ts` — legacy, frontend không gọi nữa)

Giữ tạm để có chỗ chạy PDF + slug lookup. Sẽ migrate hết sang Next.js hoặc xoá Express trong tương lai.

### Cache strategy

- `birthHash = sha256(gender|birthDate|timeIndex).slice(0,16)` — share giữa user cùng ngày sinh.
- `analyses` table: PK birthHash, lưu 6 section AI.
- `deep_readings` table: PK (birthHash, year), lưu 4 deep section (nam hiện tại đổi theo năm).
- `charts` table: 1 row per user submission; nhiều user cùng ngày sinh → nhiều chart row, nhưng share analyses + deepReadings rows.

## TypeScript + build conventions

- Backend packages (`core`, `astrology`, `ai`, `pdf`, `api`) use **NodeNext** module resolution and write **`.js` extensions in imports of `.ts` files** (e.g., `export * from './types.js';`). This is intentional — don't "fix" the extensions.
- The Next.js web app does not use NodeNext. When `apps/web` imports from `@tuvi/core` or `@tuvi/lichvannien` (which use `.js` imports internally), webpack resolves them via `next.config.js`:
  ```js
  config.resolve.extensionAlias = { '.js': ['.ts', '.tsx', '.js', '.jsx'] };
  ```
  and `transpilePackages: ['@tuvi/core', '@tuvi/lichvannien']`. If you add another `@tuvi/*` package to the web's import graph, add it to `transpilePackages` too.
- All workspace packages expose source directly: `"main": "./src/index.ts"`, `"exports": { ".": "./src/index.ts" }`. Consumers run TS source via `tsx` (bot/api) or Next's bundler (web). The `tsc` builds emit into `dist/` but nothing imports from `dist/`.

## AI package (`@tuvi/ai`)

The `openai` SDK is used here **only as an HTTP client** — `baseURL` is overridden to `https://api.deepseek.com`. It never calls api.openai.com. The `DeepseekClient` type alias + comments in `client.ts` and the `"//"` field in `package.json` document this; keep the naming consistent if you refactor.

Adding a new analysis section requires three coordinated edits:
1. `packages/core/src/types.ts` — extend `AnalysisSections` + `ANALYSIS_TITLES`
2. `packages/ai/src/prompts.ts` — add key to `SECTION_PROMPTS` + `SECTION_ORDER`
3. `apps/web/src/app/tu-vi/[slug]/page.tsx` — add key to `ANALYSIS_ORDER`

## AI concurrency + retry (`packages/ai/src/limit.ts`)

- `aiCall(fn, label)` = `pLimit(8)` semaphore + retry với exponential backoff cho 429/5xx và network error.
- Max 8 call Deepseek đồng thời từ process. Override qua env `AI_CONCURRENCY`.
- Mọi `client.chat.completions.create` PHẢI đi qua `aiCall` — `callOneSection` (markdown) và `callJsonSection` (JSON) đã wrap sẵn.
- `analyzeChart` + `analyzeDeepReadings` dùng `Promise.all` — section gọi đồng thời, progress emit theo thứ tự hoàn thành.

## PDF fonts

`@tuvi/pdf` requires `NotoSans-{Regular,Bold,Italic}.ttf` under a `fonts/` directory resolved via `getFontPaths()` (walks up from cwd). `scripts/download-fonts.ts` runs as `postinstall` — if fonts are missing at runtime the build throws "Chạy: pnpm download-fonts".
